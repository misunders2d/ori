Messaging

Connect at least one messaging platform to chat with your bot. You can set up both.

How to set up Telegram
  1. Open Telegram and search for @BotFather
  2. Send /newbot, choose a name and username for your bot
  3. BotFather will give you a token — paste it below as Telegram Bot Token
  4. That's it! Save the config, restart the server, and message your bot. It uses automatic polling — no public URL needed.
Advanced: Use webhook mode instead (requires public URL)

If your server has a public URL (Cloud Run, VPS, ngrok), you can use webhooks for lower latency:

  1. Choose any random string as a Webhook Secret and enter it below
  2. Register the webhook — open this in your browser:
    https://api.telegram.org/botTOKEN/setWebhook?url=YOUR_URL/telegram/webhook&secret_token=SECRET
  3. You should see {"ok":true}

When a Webhook Secret is configured, polling is automatically disabled.

How to set up Slack
  1. Go to api.slack.com/apps and click Create New App → From scratch
  2. Under OAuth & Permissions, add Bot Token Scopes: chat:write, channels:read, channels:history
  3. Click Install to Workspace and authorize
  4. Copy the Bot User OAuth Token (starts with xoxb-) — paste below
  5. Go to Basic Information → App Credentials, copy the Signing Secret — paste below
  6. Under Event Subscriptions, enable events and set the Request URL to your public URL + /slack/events
    Example: https://your-app.run.app/slack/events
  7. Subscribe to bot events: message.channels, message.im
  8. Save changes — your bot is now connected

Note: Slack requires a public URL for event delivery. If you're running locally, use a tunnel like ngrok http 8080.
